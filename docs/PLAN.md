# haRdQualizer — План проекту

## Context
Інтерактивний аудіо-еквалайзер/візуалізатор з 4 унікальними темами. Десктоп-додаток на Python з можливістю майбутнього портування в браузер. Джерела звуку: власний аудіо-енджин (WASAPI loopback для захоплення системного аудіо) або інтеграція зі стрімінг-сервісами.

## План 1: Десктоп-додаток

### Фаза 1 — Аудіо-енджин (з нуля)
Повністю свій пайплайн обробки звуку:

1. **`capture.py`** — обгортка над PyAudioWPatch
   - Відкриття WASAPI loopback стріму
   - Читання чанків (1024/2048/4096 семплів)
   - Callback-режим: аудіо-потік у окремому треді
   - Автодетект дефолтного аудіо-виходу

2. **`audio_buffer.py`** — кільцевий буфер
   - Thread-safe numpy масив для обміну між аудіо-тредом і UI
   - Конфігурований розмір (впливає на latency vs точність)

3. **`fft_analyzer.py`** — аналіз частот
   - `numpy.fft.rfft()` для реальних сигналів
   - Віконні функції (Hanning, Hamming, Blackman) для зменшення spectral leakage
   - Конвертація в дБ шкалу
   - Конфігурація: розмір FFT, overlap

4. **`spectrum.py`** — постобробка спектру
   - Групування FFT бінів у N смуг (логарифмічна шкала, як людське вухо)
   - Згладжування по часу (exponential decay) — бари плавно падають
   - Peak hold — пікові маркери тримаються зверху
   - Нормалізація амплітуд

5. **`beat_detector.py`** — детекція бітів
   - Відстеження різких стрибків енергії в басовому діапазоні
   - Виділення "drop" моментів (різкий перехід тиша→гучно)
   - Event-система: `on_beat`, `on_drop` → тригерять анімації у візуалізаторах

### Фаза 2 — UI каркас + базовий візуалізатор
1. **`main_window.py`** — PyQt6 вікно
   - Toolbar: перемикач тем (іконки), вибір аудіо-джерела, налаштування
   - Центральний widget — контейнер для поточного візуалізатора
   - Statusbar: поточне джерело, FPS, рівень гучності

2. **`base.py`** — абстрактний `BaseVisualizer(QWidget)`
   - `update_spectrum(bands: np.ndarray, beat: bool, drop: bool)`
   - `on_mouse_press(button, pos)`, `on_mouse_release()`, `on_mouse_move(pos)`
   - `on_mouse_drag(button, start_pos, current_pos)`
   - Кожна тема — підклас з власним рендерингом

3. **Перший робочий прототип** — прості бари на PyQtGraph BarGraphItem що рухаються під музику

### Фаза 3 — Тема 1: Dark Physics Equalizer
**Чорний фон, неонові бари, фізика взаємодії**
- PyQtGraph custom painting: градієнтні бари (синій→фіолетовий→рожевий)
- Бари реагують на частоти в реальному часі
- **ЛКМ клік** → бар "розбивається" на частинки (particles.py), анімація розльоту, потім збирається назад
- **ЛКМ зажим + рух** → "підкидання" барів вгору з фізикою гравітації, вони відскакують
- **ПКМ клік** → бар "заморожується" (кристалізація), ігнорує музику 3 сек, потім відтаює
- **ПКМ зажим + рух** → стискання/розтягування візуалізації: вправо — бари розʼїжджаються, вліво — стискаються як гармошка

### Фаза 4 — Тема 2: Cartoon Equalizer
**Яскравий, веселий, мультяшний стиль**
- Бари — желейні стовпчики з очима (QPainter малює: овальне тіло, очі, рот)
- "Особистість" за частотою: басові — товсті й сонні, середні — активні, високі — нервові/трясуться
- Танець залежить від темпу (beat_detector)
- При drop → всі бари підстрибують + хвиля
- Між барами літають бульбашки, зірочки, ноти (particle system)
- Фон — градієнт, що змінює колір від домінуючої частоти
- **ЛКМ** → бар робить гримасу (очі великі, рот О)
- **ПКМ** → бар на 3 сек перетворюється на випадковий обʼєкт (торт, ракета, квітка)

### Фаза 5 — Тема 3: Alt/Gothic Equalizer
**Готична естетика, аніме-стиль персонажі**
- Темний фон: місяць, зірки, силуети готичних будівель
- 2-3 готик-дівчини (спрайти) на передньому плані
- Еквалайзер — пурпурне полумʼя/хвиля за ними
- **При drop** → дівчата піднімають руки, волосся розвівається
- **При басах** → фон пульсує фіолетовим/червоним
- **ЛКМ на персонажі** → зміна пози: idle → dance → headbang → horns up
- **Тиха музика** → сумні, сидять; **гучна** → стоять, танцюють
- Атрибутика: шипи, чокери, темний макіяж
- Частинки: летючі миші, пелюстки троянд, іскри

### Фаза 6 — Тема 4: Retro 80s Equalizer
**Стиль старого стерео-еквалайзера 80-х** (натхнення: [easy-graphic-equalizer](https://github.com/Cioscos/easy-graphic-equalizer))
- LED-сегменти: зелений (низ) → жовтий (середина) → червоний (пік)
- Кожна смуга — стовпчик із прямокутних "лампочок" що загоряються
- Peak hold — біла/яскрава лінія зверху тримається і повільно падає
- VU-метр зліва/справа (стрілочний, як у касетниках)
- Ефекти: scanlines (CRT монітор), легке мерехтіння, glow на активних сегментах
- Фон — темно-сірий "метал" з текстурою brushed aluminum
- Шрифт — ретро цифровий (LCD/LED стиль)
- **ЛКМ на бар** → "перегорає лампочка" (один сегмент блимає і гасне, потім відновлюється)
- **ПКМ** → перемикає колірну схему LED (classic green-yellow-red / cyan-blue / amber mono)

### Фаза 7 — Інтеграція стрімінгів
- **Spotify** — Spotify Web API (OAuth2), поточний трек + audio features
- **YouTube Music** — yt-dlp для аудіо-потоку
- **SoundCloud** — SoundCloud API
- Fallback: системний loopback завжди доступний

### Фаза 8 — Polish
- Налаштування: чутливість, кількість смуг, кольори, розмір вікна
- Локалізація uk/en (gettext .po файли)
- Збереження налаштувань (JSON конфіг)
- Гарячі клавіші: `1/2/3/4` — теми, `Space` — пауза, `S` — налаштування
- Іконка в системному треї
- Повноекранний режим (F11)

## План 2: Браузерна версія (майбутнє)

### Підхід
Окремий фронтенд на **JavaScript/TypeScript**. Візуальна логіка портується, аудіо-енджин замінюється на Web Audio API.

### Архітектура
- **Web Audio API** → `AnalyserNode` замість нашого `fft_analyzer.py`
- **Canvas 2D / WebGL (PixiJS)** → замість PyQtGraph
- **React або Vanilla JS** — UI оболонка
- **Spotify Web Playback SDK** — грає прямо в браузері (потребує Premium)

### Етапи
1. Порт `spectrum.py` → JS (AnalyserNode + свій постпроцесинг)
2. Порт beat_detector → JS
3. Порт кожного візуалізатора (Dark → Cartoon → Gothic → Retro)
4. Spotify Web Playback SDK інтеграція
5. PWA обгортка (offline, install prompt)

### Обмеження
- Немає WASAPI loopback → тільки мікрофон або tab audio capture (Chrome extension)
- Spotify Web Playback SDK → тільки Premium
- Менша продуктивність для складних particle systems
